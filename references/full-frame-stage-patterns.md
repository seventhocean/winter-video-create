## 通用规则

模式只规定空间骨架，不规定具体文案、颜色或字体；颜色和运动从 `styleProfile` 读取。

普通解释舞台使用 3–5 个空间锚点：

```text
top-left       top-right
center-left    center-right
lower-left     lower-right
```

人物脸、嘴和字幕区属于避让区。可以让细路径经过人物背景，但大型文字卡、主节点和真实 UI 不得长时间覆盖眼睛、嘴或字幕。

每个模式都要写入：

```json
{
  "layoutProfile": "full-frame-explainer",
  "spatialAnchors": ["top-left", "center-right", "lower-left"],
  "topology": "curved-flow",
  "stateChange": "node-connected"
}
```

## 仪表盘舞台

- `topology`：`dashboard`
- 适合：多个指标、能力边界、当前状态和最终结论。
- 主视觉：中心或偏左的大数字 / 仪表盘。
- 辅助元素：右上标签、左下基线、右下结论卡、环形进度或状态灯。
- 动画顺序：刻度 / 背景轨 → 数字 → 环形进度 → 状态标签 → 结论强调。
- 状态变化：数值增长、指针移动、负载从低到高或状态由 pending 变为 ready。
- 注意：不要用仪表盘装饰没有数字依据的结论。

## 曲线路径舞台

- `topology`：`curved-flow`
- 适合：输入 → 处理 → 输出、API 调用、工作流和因果关系。
- 主视觉：一条从左上或左侧开始的路径。
- 辅助元素：输入节点、处理中间节点、结果节点、关键词标签和方向箭头。
- 动画顺序：路径生长 → 起点接通 → 中间节点亮起 → 终点完成 → 结果标签落位。
- 状态变化：连接完成、权限通过、数据到达或错误节点被替换。
- 注意：曲线必须表达方向，不能只是背景线。

## 资料阶梯舞台

- `topology`：`ladder` 或 `stack`
- 适合：知识库、上下文、素材累积、信息从底层到结论的过程。
- 主视觉：由下至上的阶梯或资料堆。
- 辅助元素：来源标签、层级名称、当前高亮层、最终输出卡。
- 动画顺序：底层资料进入 → 层级累积 → 当前层高亮 → 上方结论生成。
- 状态变化：信息从分散变为结构化，或重要信息从堆叠中被提取。
- 注意：每一级只表达一个层次，不要把整段文章缩成小字堆在阶梯上。

## 工具网络舞台

- `topology`：`network`
- 适合：Agent、Skill、工具链、权限和系统组成。
- 主视觉：中心任务节点或 Agent 节点。
- 辅助元素：四周工具节点、连接线、权限标签、输入输出状态。
- 动画顺序：中心节点出现 → 工具节点分批出现 → 连线接通 → 当前调用路径高亮。
- 状态变化：工具被授权、连接建立、节点被替换或任务完成。
- 注意：节点数量超过 7 个时必须分批或分组，否则会变成噪声。

## 诊断轨道舞台

- `topology`：`diagnostic-rail`
- 适合：问题排查、清单、决策树、错误原因和多条判断路径。
- 主视觉：上下或左右排列的诊断轨道。
- 辅助元素：检查点、错误标记、通过标记、当前指针、最终建议。
- 动画顺序：轨道建立 → 检查点逐项亮起 → 错误 / 通过状态出现 → 指针指向结论。
- 状态变化：unknown → checking → passed / blocked。
- 注意：诊断轨道必须有顺序或决策关系，不能只是纵向罗列文字。

## 左右对比舞台

- `topology`：`split-compare`
- 适合：改造前后、低效 / 高效、信息不足 / 信息完整、旧流程 / 新流程。
- 主视觉：左右两种状态，中央一条关系线。
- 辅助元素：差异标签、被划掉的旧状态、被强调的新状态、底部结论。
- 动画顺序：旧状态进入 → 问题点标记 → 中央箭头完成 → 新状态进入 → 结论强调。
- 状态变化：旧状态被替换、指标反转或缺失信息被补齐。
- 注意：两侧必须形成真正对比，不能只是两个无关面板并排。

## 素材 PIP 舞台

- `topology`：`asset-callout`
- 适合：用户提供的截图、手机录屏、网页、产品画面和案例素材。
- 主视觉：人物或背景画面中的真实素材框。
- 辅助元素：素材来源标签、局部放大框、指向线、解释性结论。
- 动画顺序：章节标签 → 素材框进入 → 局部焦点移动 → 解释标签落位 → 素材退出或放大为全屏。
- 状态变化：素材从预览变为重点、局部区域被确认或操作结果出现。
- 注意：真实 UI 中的按钮、输入框、光标和结果区域必须优先保护；如果是实操段，遵守 `operation-upper`。
